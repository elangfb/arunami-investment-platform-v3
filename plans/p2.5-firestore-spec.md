# P2.5 — Doc / Drive / AI / Audit subsystem → Firestore (full parity)

Reconciled from the `p2_5-docs-drive-design` design workflow (8 agents) + existing P2 layout.
Pattern: same dispatcher seam as P2 — `X.ts` routes by `DATA_BACKEND` to `X.prisma.ts` / `X.firestore.ts`
via `dispatchRead`/`dispatchWrite`. Reuse `firebase/collections.ts`, `repo/doc-ids.ts`,
`firebase/timestamps.ts`, `repo/errors.ts`, `firebase/firestore.ts`. Verify each module on the emulator.

## Reconciliations (override design-agent proposals)
- **DecisionCheckpoint** lives **top-level** `decisionCheckpoints` (P2's read already does
  `where applicationId == … orderBy createdAt desc limit 1`; composite index already in firestore.indexes.json).
  Only the WRITE side is missing. NOT a subcollection.
- **DocumentVersion**, **ExtractionRun**, **ApplicationDocumentFill** → subcollections under
  `applications/{appId}/…` (consistent with P2's documents/history/etc.), NOT under docLinkages.
- **ApplicationDocumentFill** doc-id = `${slug(docId)}__${slug(tokenName)}` (appId is the parent path)
  → enforces the (appId,docId,tokenName) uniqueness via the composite doc-id.

## New composite indexes (firestore.indexes.json)
- `applications/{appId}/extractionRuns` (collection-group or scoped): `ok ASC, createdAt DESC` (latestOk query)
- `docAccessGrant`: `docId ASC, role ASC` (reconcileFrozenDocGrants writers query)
- `aiInteraction`: `applicationId ASC, createdAt DESC`; `userId ASC, createdAt DESC`
- `impersonationAudit`: `superadminId ASC, startedAt DESC`
- `applications`: `disbursedAt ASC, applicationStatus ASC` (review-due-notices) — verify against Firestore != null handling
- (DocumentVersion list = orderBy createdAt desc within subcollection → single-field, auto)

## Implementation order (dependency-driven)
1. **Shared infra** — collections.ts (new COL/SUB), doc-ids.ts (docAccessGrantId, driveRootGrantId,
   templateReferenceTextId, documentFillId), errors if any. fs-test-helpers TOP_LEVEL + indexes.
2. **decisionCheckpoint WRITE** — new `repo/checkpoint.{ts,prisma,firestore}.ts` (or extend service);
   write top-level matching P2's read fields (applicationId, decision, muap/rskDocId, storageKeys,
   sizes, contentHash, decidedAt, riskPolicy snapshot, exploredSources, createdAt). PDF bytes → GCS.
3. **docLinkage** — `repo/doc-linkage.{ts,prisma,firestore}.ts` (the hub: get/upsert/update fields).
4. **documentVersion** — snapshot create + listDocumentVersions + read-one (rollback).
5. **extractionRun** — create + getApplicationDocs (latest + latestOk).
6. **applicationDocumentFill** — sync-v2 reads/updates + lost-in-doc.
7. **docAccessGrant** — ensureGrant upsert + reconcile downgrade.
8. **driveRef + driveRootGrant** — root folder ref + per-email grant ledger.
9. **templateReferenceText** — get/getBulk/upsert/count (dormant but parity).
10. **aiInteraction** — recordAiInteraction (append, best-effort).
11. **impersonationAudit** — create + stop (updateMany→query+batch).
12. **Action-layer cleanups** — replace direct prisma reads with repo calls:
    - application-create.core: customer.findUnique→getCustomer; application.count→backend-aware next-display-id (count() or counter).
    - application-review.core / lineage-read / context-layers: application/customer reads → repo fns.
    - discovery-actions.core + discover.ts: app+customer driveFolderId read + app.update(driveFolderId) → repo fns.
    - approval-notices: approvalRoutingRule.findMany → backend-aware listAllApprovalRoutingRules.
    - review-due-notices: application.findMany(disbursedAt!=null, status!=closed)+customer.reviewCadenceMonths → backend-aware.
    - health: prisma.$queryRaw → backend-aware liveness (firestore: getDb().collection('applications').limit(1).get()).
