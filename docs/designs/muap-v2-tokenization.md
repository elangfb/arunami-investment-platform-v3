## Final taxonomy

Conventions that survived all revs (encode these in the build-time registry):

| Cat | Meaning | Token convention |
|---|---|---|
| A | Fact (single value) | `{{name}}` |
| B | Categorical | binary → `{{name_or_X}}`; 3+ values → `{{name}}` |
| C | Narrative (AI prose) | `{{ai_name}}` — single canonical prefix; no `ain_*`, no bare `uraian_*` |
| D | Composite | decompose into multiple tokens |
| E | Presentational | leave alone (no token) |
| F | Pseudo-blank (`-`, `—`) | case-by-case |

Other rules:

- `snake_case`. `_1` / `_2` suffix for fixed-row tables.
- Reuse over duplication — same value across sections → one NamedRange, multiple placements (e.g. `plafond` reused in T6, T56, T83).
- **AI may only write `ai_*` slots.** All other tokens are deterministic fills from app data or human-confirmed input. The masking/scrub pipeline (`server/ai/narrative.ts` + `pii-mask.ts`) is the enforcement seam.
- **Indonesian enum values** where the Doc is rendered for the bank reviewer (e.g. `rekomendasi_analis` ∈ `SETUJU / SETUJU DENGAN SYARAT / TOLAK`). Map to English `Approve/Conditional/Reject` at the code boundary — see memory `decision-verbs-english`.

## Pitfalls (preserve these in the playbook)

1. **Bracket scan cap.** The diagnostic regex was `\[[^\[\]\n]{3,200}\]`; MUAP narrative guidance runs 200–400 chars and was silently dropped. Always cap ≥1500 and verify the diagnostic surfaces the long-form narratives.
2. **PDF visual inspection is non-optional.** Scan order ≠ section order; the table at scan-index N may be a different semantic table than the section name suggests (T43 / T45 were swapped in the rev-1 inference). Always cross-check against a PDF render of the references doc before naming.
3. **Polarity bugs in binaries.** `persen_non_halal_income` vs `persen_halal_income`; `gharar` (T61, contract-structure) vs `pelanggaran_syariah` (T63, substantive). Read the row label, don't infer from the column header.
4. **Misnamed enums.** `is_tambahan_line_facility` looked binary in scan, but the PDF has 3 checkboxes → `kategori_fasilitas` ∈ `Baru / Perpanjangan / Tambahan`. Count the checkboxes in the PDF, not the scan.
5. **Section title ≠ field intent.** "T70 Total Angsuran" was a section-name guess from rev 1; the actual cell is `biaya_administrasi_pembiayaan_flat`. Read the field label, not the section heading.
6. **Single prefix per kind.** `ain_*` is a typo; `uraian_*` is a sibling-convention drift. Enforce `ai_*` for all AI-narrative tokens at the registry build-time gate.
7. **Reuse is cheaper than parallel tokens.** Same value in two sections → one NamedRange, two placements (`plafond`, `nama_perusahaan`, `sccr_persen`, `equivalent_pa_realisasi`). The Doc API supports it; the build-time registry should refuse duplicate concepts.
8. **Hijra templates are literal source of truth.** When RSK refers to a field MUAP doesn't have, **RSK owns it** (`rsk_*`). Three times during this work an agent tried to "promote" RSK-side fields to MUAP. The build-time registry must fail-loud on R1 `[muap]` claims that don't resolve to an existing MUAP token. See `rsk-v2-tokenization.md` rev 2.1 audit for the 16-token reclassification.
